\doxysubsection{Detailed Description}
Classe base para todas as procuras. 

Esta classe base para procuras... (separação de TProcuras\+Construtivas, em curso). pretende-\/se que depois as sub-\/classes tipo TProcura\+Melhorativa, possa herdar de \doxylink{classTProcura}{TProcura}, não ficando com partes da procura construtiva que são desnecessárias

{\bfseries{Observação\+:}} Alguns métodos e parâmetros terão efeito apenas se determinados métodos forem redefinidos na subclasse.

{\footnotesize\ttfamily void TProcura\+::\+Debug (\begin{DoxyParamCaption}\item[{void}]{ }\end{DoxyParamCaption})\hspace{0.3cm}{\ttfamily [virtual]}}



Mostra o estado no ecrã, para debug. 

\begin{DoxyNote}{Note}
Redefinição opcional. Necessário para visualizar a procura, e explorar o espaço manualmente.
\end{DoxyNote}
Esta função deverá mostrar claramente o estado atual, em texto mas da forma mais confortável possível. O formato texto destina-\/se principalmente a quem implementa o problema, e não utilizadores finais. É importante poder explorar o espaço de estados, para verificar a correta implementação dos sucessores, como também possa ver a árvore de procura dos algoritmos, para árvores pequenas, e assim detectar bugs.

\begin{DoxyNote}{Note}
Antes de cada linha, chame a função Nova\+Linha(). Dependendo do contexto, {\ttfamily Nova\+Linha()} pode imprimir caracteres que representam os ramos da árvore de procura, criando uma visualização textual que simula a estrutura da procura.

A exibição do estado pode variar conforme o nível de debug definido em {\ttfamily parametro\mbox{[}nivel\+Debug\mbox{]}.valor}. Um nível menor pode mostrar informações mais sucintas, enquanto um nível maior pode detalhar todas as variáveis do estado.
\end{DoxyNote}

Se {\ttfamily configuracoes} estiver vazia, o teste empírico será executado apenas com a configuração atual, avaliando seu desempenho isoladamente, sem comparação com outras configurações.

Inicializa a interação com o utilizador. 

\begin{DoxyNote}{Note}
Redefinição necessária para definir as instancias existentes.
\end{DoxyNote}
Esta função arranca com o teste manual, orientada para o programador. A interface permite\+:
\begin{DoxyItemize}
\item visualizar e trocar de instância
\item explorar o espaço de estados nessa instancia, executando ações
\item ver um caminho que esteja gravado (por exploração manual ou por execução de um algoritmo)
\item ver e editar qualquer parametro de execução
\item o algoritmo é também um parametro, podendo naturalmente ser alterado
\item há parametros sobre limites de execução, informação de debug, opções de implementação e opções de algoritmos
\item executar o algoritmo com a configuração atual
\item adicionar a configuração atual a um conjunto de configurações de teste
\item executar um teste empírico, executando todas as configurações de teste, no conjunto de instâncias selecionadas
\end{DoxyItemize}

Esta função deve ser redefinida para inicializar a variável com informação dos IDs das instâncias disponíveis. Essa variável é do tipo TParametro, mas não está na lista de parametros, devendo ser inicializada aqui.

\begin{DoxyNote}{Note}
A instância selecionada irá ser carregada em \doxylink{classTProcura_ab3d1e667ee7b23575acd8c83fad1c751}{Solucao\+Vazia()}, utilizando o valor atual. 

Esta função deve ser o ponto de entrada, a executar no main.
\end{DoxyNote}
